## 二、技术栈决策

### 2.1 数据库：PostgreSQL 单数据源

**结论：PostgreSQL 作为唯一数据库，替换 MySQL**

**采用理由：**
- interview-guide 的向量能力完全依赖 Spring AI 的 `PgVectorStore`，切换至 MySQL 需要自研向量检索
- MySQL 8.0 官方尚无稳定的向量索引支持（`VECTOR` 类型在 9.x 才有），生产可靠性不足
- pgvector 已是向量存储的事实标准，Spring AI 内置支持，迁移成本极低
- **单数据源**：业务表（MyBatis-Plus）和 `vector_store`（Spring AI 自动管理）共用同一个 PostgreSQL 实例，无需双数据源配置

**数据划分：**

```
PostgreSQL（唯一数据源）
├── 业务表（MyBatis-Plus 管理）
│   ├── t_llm_provider, t_llm_global_setting
│   ├── t_resume, t_resume_analysis
│   ├── t_interview_session, t_interview_answer
│   ├── t_knowledge_base, t_rag_chat_session, t_rag_chat_message
│   ├── t_interview_schedule
│   └── t_voice_session, t_voice_message, t_voice_evaluation
└── vector_store（Spring AI PgVectorStore 自动管理，HNSW 索引，1024 维 COSINE）
```

### 2.2 异步消息：Redis Stream

**结论：暂定 Redis Stream，不使用 RocketMQ**

| 维度 | Redis Stream | RocketMQ |
|------|-------------|---------|
| **部署成本** | 复用现有 Redis | 需额外部署 NameServer + Broker |
| **消息可靠性** | 支持 ACK，持久化（appendonly） | 更强，支持事务消息、精确一次 |
| **延迟消息** | 不支持 | 支持 |
| **消息回溯** | 支持（消费者组 XREADGROUP） | 支持 |
| **与 Spring AI 整合** | 无耦合 | 无耦合 |
| **适用规模** | DAU < 5000 | 大规模生产 |

**升级信号**：DAU > 5000 或需要延迟消息/事务消息时，切换至 RocketMQ。

**三个异步管道：**
1. `resume-analysis-stream`：简历 AI 评分
2. `interview-evaluate-stream`：面试答题评估
3. `knowledge-vectorize-stream`：知识库文档向量化

### 2.3 Java 版本：升级至 21（虚拟线程）

**结论：必须升级，成本近乎为零**

```yaml
# application.yml
spring:
  threads:
    virtual:
      enabled: true
```

**理由：** LLM 调用是纯 IO 密集型操作，虚拟线程让每次 AI 请求无需占用平台线程，吞吐量提升显著。Spring Boot 3.2+ 已内置支持，只需一行配置。

### 2.4 Spring AI 版本：1.0.0（兼容 Spring Boot 3.x）

`interview-ai` 模块（`libs/interview-ai`）已引入 Spring AI 1.1.4，使用新式 artifact 名：
- `spring-ai-starter-model-openai`（OpenAI 兼容协议，覆盖 DashScope 等所有 Provider）
- `spring-ai-starter-vector-store-pgvector`（pgvector 向量存储）

interview-guide 使用 Spring AI 2.0，目标项目使用 **1.1.4**，主要 API 差异：

| 功能 | Spring AI 2.0（interview-guide） | Spring AI 1.1.4（mhd-interview） |
|------|----------------------------------|----------------------------------|
| ChatClient 创建 | `ChatClient.builder(model)` | `ChatClient.builder(chatModel)` |
| 向量存储 | `PgVectorStore` | `PgVectorStore`（相同） |
| 结构化输出 | `BeanOutputConverter` | `BeanOutputConverter`（相同） |
| ToolCallingManager | 存在 | **不存在**，迁移时移除 |

**interview-ai 模块职责**（公共 AI 能力库，`libs/interview-ai`）：
- `StructuredOutputInvoker`（已有，保留并完善）
- `LlmProviderRegistry`（新增：多 Provider 动态路由）
- `PromptSanitizer`（新增：Prompt 注入防护）

## 三、包结构重新设计

### 3.1 interview-guide 包结构问题分析

| 问题 | 具体表现 | 改进方案 |
|------|---------|---------|
| **Service 类过多** | `VoiceInterviewService` 内包含 ASR、TTS、评估、会话四类逻辑 | 拆分为独立 Service 类，各司其职 |
| **全局枚举单文件** | `AsyncTaskStatus`、`LlmProvider` 等枚举散落各处或集中在一个大枚举文件 | 按模块归属放置枚举，`common` 中只保留跨模块枚举 |
| **缺少 Service 接口** | 直接用 `@Service` 实现类，无接口层 | 所有 Service 增加接口，`impl` 子包存放实现 |
| **包名冗长** | `modules/knowledgebase/`、`modules/voiceinterview/`、`modules/interviewschedule/` | 简化为 `module/knowledge/`、`module/voice/`、`module/schedule/` |
| **Mapper 层缺失** | 直接用 JPA Repository，命名混乱 | 改用 MyBatis-Plus `BaseMapper`，统一命名 `XxxMapper` |

### 3.2 新包命名规则

| 旧包名（interview-guide） | 新包名（mhd-interview） |
|--------------------------|------------------------|
| `modules/resume/` | `modules/resume/` |
| `modules/interview/` | `modules/interview/` |
| `modules/knowledgebase/` | `modules/knowledge/` |
| `modules/interviewschedule/` | `modules/schedule/` |
| `modules/voiceinterview/` | `modules/voice/` |
| `modules/llmprovider/` | `modules/llmprovider/`（保留完整名称，清晰表达 LLM Provider） |
| `infrastructure/` | `infrastructure/`（保留） |
| `common/ai/` | `interview-ai` 模块（`libs/interview-ai`） |

## 四、Service 层设计

### 4.1 接口+实现模式

所有业务 Service 必须遵循接口+实现模式：

```
service/
├── XxxService.java          # 接口（public interface）
└── impl/
    └── XxxServiceImpl.java  # 实现（@Service）
```

**采用理由：**
- 与 mhd-interview 现有代码风格一致
- 便于单元测试 Mock
- 为后续 AOP 扩展（事务、缓存、限流）预留代理入口

### 4.2 各模块 Service 拆分

| 模块 | Service 接口 | 职责 |
|------|-------------|------|
| **provider** | `ProviderConfigService` | Provider CRUD、加密、缓存刷新 |
| **resume** | `ResumeService` | 简历上传触发、列表、详情、删除 |
| | `ResumeAnalysisService` | AI 评分、异步管道、历史查询、PDF 导出 |
| **interview** | `InterviewSessionService` | 会话创建、状态机、历史 |
| | `InterviewQuestionService` | AI 出题、答题提交、评估触发 |
| **knowledge** | `KnowledgeBaseService` | 文档管理（上传、列表、删除） |
| | `KnowledgeVectorService` | 向量化异步管道、pgvector 操作 |
| | `RagChatService` | RAG 查询、聊天会话管理 |
| **schedule** | `InterviewScheduleService` | 日程 CRUD、AI 解析邀请 |
| **voice** | `VoiceSessionService` | 语音会话生命周期管理 |
| | `VoiceEvaluationService` | 多轮评估异步管道 |

## 附录：MybatisPlusMetaObjectHandler

所有 Entity 自动填充 `createTime`、`updateTime`，需在 `interview-business` 中配置：

```java
@Component
public class InterviewMetaObjectHandler implements MetaObjectHandler {

    @Override
    public void insertFill(MetaObject metaObject) {
        LocalDateTime now = LocalDateTime.now();
        this.strictInsertFill(metaObject, "createTime", LocalDateTime.class, now);
        this.strictInsertFill(metaObject, "updateTime", LocalDateTime.class, now);
    }

    @Override
    public void updateFill(MetaObject metaObject) {
        this.strictUpdateFill(metaObject, "updateTime", LocalDateTime.class, LocalDateTime.now());
    }
}
```

Entity 字段注解配合：

```java
@TableField(value = "create_time", fill = FieldFill.INSERT)
private LocalDateTime createTime;

@TableField(value = "update_time", fill = FieldFill.INSERT_UPDATE)
private LocalDateTime updateTime;
```
